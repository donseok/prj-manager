## 1. 시작하기

### 1.1 시스템 요구사항

| 항목 | 권장 사양 |
|------|-----------|
| 브라우저 | Chrome, Edge, Safari 최신 버전 |
| 해상도 | 1280×720 이상 (1920×1080 권장) |
| 네트워크 | 인터넷 연결 필요 (Supabase 연동 시) |

### 1.2 로그인

1. 브라우저에서 DK Flow 접속 URL을 엽니다.
2. **로그인** 탭을 선택합니다.
3. 이메일과 비밀번호를 입력한 후 **로그인** 버튼을 클릭합니다.

> 💡 **로컬 모드**: Supabase가 설정되지 않은 환경에서는 자동으로 로컬 모드로 접속됩니다. 로컬 모드에서는 별도의 계정 없이 바로 사용할 수 있습니다.

### 1.3 회원가입

1. 로그인 화면에서 **회원가입** 탭을 선택합니다.
2. **이름**, **이메일**, **비밀번호** (6자 이상)를 입력합니다.
3. **회원가입** 버튼을 클릭합니다.
4. 온라인 모드(Supabase)에서는 **관리자 승인 대기(pending)** 상태가 됩니다. 관리자가 승인을 완료하면 정상적으로 이용할 수 있습니다.

> 💡 **로컬 모드**에서는 승인 과정 없이 즉시 로그인되어 홈 화면으로 이동합니다.

## 2. 홈 화면

홈 화면은 시스템 진입 후 가장 먼저 보이는 메인 대시보드입니다.

### 2.1 요약 카드

상단 히어로 영역에서 다음 정보를 확인할 수 있습니다:

| 카드 | 설명 |
|------|------|
| **전체 프로젝트** | 시스템에 등록된 모든 프로젝트 수 |
| **진행중** | 현재 `진행중` 상태인 프로젝트 수 |
| **최근 기록** | 빠르게 접근할 수 있는 최근 프로젝트 수 |

### 2.2 주요 액션

- **새 프로젝트 시작** → 프로젝트 생성 화면으로 이동
- **전체 프로젝트** → 프로젝트 목록 페이지로 이동

### 2.3 지표 패널

- **Active Ratio**: 전체 프로젝트 대비 진행중 프로젝트 비율
- **Completed**: 완료된 프로젝트 수
- **Quick Access**: 최근 프로젝트 바로가기 (최대 3개)

### 2.4 최근 프로젝트 섹션

하단에 최근 작업한 프로젝트 카드가 표시됩니다. 카드를 클릭하면 해당 프로젝트의 대시보드로 바로 진입합니다.

## 3. 프로젝트 관리

### 3.1 프로젝트 목록

사이드바의 **전체 프로젝트** 또는 홈 화면의 **프로젝트 라이브러리** 카드를 클릭하면 모든 프로젝트를 카드 형태로 확인할 수 있습니다.

- 각 카드에 프로젝트명, 상태(준비/진행중/완료), 기간이 표시됩니다.
- 카드를 클릭하면 프로젝트 대시보드로 진입합니다.

### 3.2 프로젝트 생성

1. 홈 화면의 **새 프로젝트 시작** 버튼 또는 사이드바의 **+** 버튼을 클릭합니다.
2. 프로젝트 기본 정보를 입력합니다:
   - **프로젝트명** (필수)
   - **설명**
   - **시작일** / **종료일**
3. 생성 후 자동으로 프로젝트 대시보드로 이동합니다.

### 3.3 프로젝트 상태

프로젝트는 다음 3가지 상태를 갖습니다:

| 상태 | 설명 |
|------|------|
| **준비** | 프로젝트 준비 단계 (기본 상태) |
| **진행중** | 프로젝트가 활발하게 진행 중 |
| **완료** | 프로젝트 완료 (완료일 자동 기록) |

> ⚠️ 상태 변경은 **관리자 권한**이 있는 사용자만 가능합니다.

## 4. 대시보드

프로젝트 진입 시 가장 먼저 보이는 화면으로, 프로젝트의 전체 현황을 한눈에 파악할 수 있습니다.

### 4.1 상단 현황 영역

| 항목 | 설명 |
|------|------|
| **전체 작업** | 등록된 리프(말단) 작업 수 |
| **멤버** | 프로젝트 참여 멤버 수 |
| **지연** | 계획 종료일을 초과한 작업 수 |

### 4.2 진행률 카드

- **Actual Progress**: 전체 실적 공정율 (프로그레스 바 + 백분율)
- **계획 공정율**: 계획 대비 진행률 비교

### 4.3 작업 현황 카드

| 카드 | 설명 |
|------|------|
| **진행중 작업** | 현재 진행 중인 작업 수 |
| **완료 작업** | 완료된 작업 수 |
| **리스크 작업** | 지연 중인 작업 수 |

### 4.4 차트 섹션

| 차트 | 설명 |
|------|------|
| **상태별 분포** | 대기/진행중/완료/보류 작업 비율 |
| **담당자별 진행률** | 각 담당자의 작업 완료/잔여 현황 (막대 그래프) |
| **Phase별 진행률** | 단계별 계획 vs 실적 공정율 비교 (막대 그래프) |
| **프로젝트 일정 요약** | 시작일, 종료일, 일정 경과율, 총/경과/잔여일 |
| **Phase 가중치 분포** | 각 Phase의 가중치 비중 (도넛 차트) |

### 4.5 작업 큐

| 큐 | 설명 |
|----|------|
| **지연 작업** | 지연 일수와 함께 지연된 작업 목록 |
| **금주 작업** | 이번 주에 진행 예정인 작업 |
| **차주 작업** | 다음 주에 진행 예정인 작업 |
| **최근 완료 작업** | 가장 최근에 완료된 작업 목록 |

### 4.6 현황 보고서 내보내기

대시보드 상단의 **현황 보고서** 버튼을 클릭하면 Word(.docx) 형식의 프로젝트 현황 보고서를 자동 생성하여 다운로드합니다.

## 5. WBS (작업분류체계)

WBS 페이지에서는 엑셀과 유사한 테이블 형태로 프로젝트의 작업을 계층적으로 관리합니다.

### 5.1 작업 계층 구조

| 레벨 | 구분 | 설명 |
|------|------|------|
| Level 1 | **Phase** | 프로젝트 단계 (분석, 설계, 개발, 테스트 등) |
| Level 2 | **Activity** | 단계 내 활동 |
| Level 3 | **Task** | 구체적 작업 |
| Level 4 | **Function** | 세부 기능/산출물 |

### 5.2 작업 추가

- **Phase 추가**: 상단 도구 모음의 **Phase 추가** 버튼 클릭
- **하위 작업 추가**: 각 작업 행 오른쪽의 **+** 버튼 클릭

### 5.3 인라인 편집 (엑셀 스타일)

테이블의 각 셀을 클릭하면 바로 편집할 수 있습니다:

| 컬럼 | 편집 방식 |
|------|-----------|
| **작업명** | 클릭 후 텍스트 입력 → Enter 또는 영역 밖 클릭으로 확정 |
| **산출물** | 클릭 후 텍스트 입력 |
| **담당자** | 드롭다운에서 멤버 선택 |
| **가중치** | 클릭 후 숫자 입력 (소수점 지원) |
| **계획시작/종료** | 날짜 선택기로 입력 |
| **실적시작/종료** | 날짜 선택기로 입력 |
| **계획/실적 공정율** | 클릭 후 0~100 숫자 입력 |
| **상태** | 드롭다운에서 선택 (대기/진행중/완료/보류) |

### 5.4 작업 상태

| 상태 | 설명 |
|------|------|
| **대기** | 아직 시작하지 않은 작업 |
| **진행중** | 현재 수행 중인 작업 |
| **완료** | 완료된 작업 |
| **보류** | 일시 중단된 작업 |

### 5.5 도구 모음

| 기능 | 설명 |
|------|------|
| **Phase 추가** | 최상위 Phase 작업 추가 |
| **전체 펼침** | 모든 작업 트리 확장 |
| **전체 접기** | 모든 작업 트리 축소 |
| **엑셀 다운로드** | WBS를 엑셀(.xlsx) 파일로 내보내기 |
| **되돌리기 (Undo)** | 마지막 변경 취소 |
| **다시하기 (Redo)** | 취소한 변경 복원 |

### 5.6 자동화 기능

WBS 페이지 상단에서 다양한 자동화 도구를 사용할 수 있습니다.

#### 템플릿으로 시작

WBS 상단의 **템플릿** 버튼을 클릭하면 프로젝트 유형별 기본 WBS 구조를 자동으로 생성할 수 있습니다. 프로젝트 유형(소프트웨어 개발, 건설, 제조 등)을 선택하면 해당 유형에 맞는 Phase, Activity, Task가 자동으로 구성됩니다.

#### 빠른 초안 생성

**빠른 초안** 버튼을 클릭한 후, 프로젝트에 대한 텍스트 설명을 입력하면 WBS 초안이 자동으로 생성됩니다. 간단한 프로젝트 개요만으로도 작업 분류 체계의 초안을 빠르게 만들 수 있습니다.

#### 자동 일정 계산

**자동 일정** 버튼을 클릭하면 각 작업의 기간(duration)과 선행작업 관계를 기반으로 일정이 자동으로 산출됩니다. 프로젝트 시작일로부터 순차적으로 일정이 배정됩니다.

#### 자동 가중치 배분

**가중치 자동배분** 버튼을 클릭하면 각 작업의 기간을 기준으로 Phase별 가중치가 자동으로 계산됩니다. 기간이 긴 작업일수록 높은 가중치가 배분됩니다.

#### 컨텍스트 메뉴 (우클릭)

WBS 테이블의 행을 우클릭하면 다음 메뉴가 표시됩니다:
- **같은 레벨 추가**: 선택한 행과 동일 레벨의 작업 추가
- **하위 작업 추가**: 선택한 행의 하위 작업 추가
- **행 복제**: 선택한 행을 복사하여 새 행 생성
- **행 삭제**: 선택한 행 및 하위 작업 삭제
- **복사**: 선택한 행 정보 복사

### 5.7 자동 저장

WBS에서의 모든 변경사항은 **자동 저장**됩니다. 별도의 저장 버튼 없이 입력 후 약 0.7초 뒤 자동으로 서버에 반영됩니다.

## 6. 간트 차트

간트 차트 페이지에서는 작업 일정을 시각적 타임라인 형태로 확인하고 관리합니다.

### 6.1 화면 구성

| 영역 | 설명 |
|------|------|
| **상단 히어로** | 표시 작업 수, 오픈(미완료) 작업 수, 지연 작업 수 |
| **작업 포커스 카드** | 선택한 작업의 상세 정보 (기간, 공정율, 지연 등) |
| **필터/검색 바** | 작업 검색, 상태 필터, 뷰 옵션 설정 |
| **간트 차트** | 좌측 작업 목록 + 우측 타임라인 바 |

### 6.2 검색 및 필터

| 기능 | 설명 |
|------|------|
| **검색** | 작업명, 산출물, 담당자로 실시간 검색 |
| **전체** | 모든 작업 표시 |
| **진행중/대기** | 완료되지 않은 작업만 필터 |
| **지연** | 지연된 작업만 필터 |
| **완료** | 완료된 작업만 필터 |

### 6.3 뷰 옵션

| 옵션 | 설명 |
|------|------|
| **보기 범위** | 6주 / 12주 / 24주 중 선택 |
| **행 밀도** | Compact (조밀) / Comfortable (여유) |
| **주말 강조** | 타임라인에서 주말 영역 강조 On/Off |

### 6.4 작업 선택

- 좌측 목록에서 작업을 클릭하면 **포커스 카드**에 상세 정보가 표시됩니다.
- 차트 영역이 해당 작업으로 자동 스크롤됩니다.

### 6.5 차트 범례

| 색상 | 의미 |
|------|------|
| 🟢 계획 바 (진한 청록) | 계획 일정 범위 |
| 🟩 실적 바 (밝은 초록) | 실적 일정 범위 |
| 🔴 오늘선 (빨간 세로선) | 현재 날짜 |

### 6.6 엑셀 다운로드

필터/검색 영역의 **엑셀 다운로드** 버튼으로 현재 필터 기준의 간트 데이터를 엑셀로 내보낼 수 있습니다.

## 7. 멤버 관리

프로젝트에 참여하는 팀원을 관리하는 페이지입니다.

### 7.1 멤버 추가

1. **멤버 추가** 버튼을 클릭합니다.
2. 팝업에서 **이름**과 **역할**을 입력합니다.
3. **추가** 버튼을 클릭합니다.

### 7.2 역할 유형

| 역할 | 권한 |
|------|------|
| **소유자** | 모든 권한 (프로젝트 생성자) |
| **관리자** | 대부분의 관리 권한 |
| **멤버** | 작업 수행 및 업데이트 |
| **뷰어** | 조회만 가능 |

### 7.3 멤버 정보 편집

- **이름 수정**: 멤버 카드의 **연필 아이콘(✏️)** 클릭 → 수정 → Enter 또는 **✓** 클릭
- **역할 변경**: 멤버 카드 하단의 **역할 지정** 드롭다운에서 변경
- **멤버 삭제**: **휴지통 아이콘(🗑️)** 클릭 → 확인

### 7.4 통계 카드

| 카드 | 설명 |
|------|------|
| **Team Size** | 등록된 전체 멤버 수 |
| **Admins** | 관리 권한 사용자 수 (소유자 + 관리자) |
| **Contributors** | 실무 작업 참여자 (멤버 역할) 수 |

## 8. 근태 관리

프로젝트 멤버의 근무 상태를 기록하고 관리하는 페이지입니다.

### 8.1 근태 페이지 접근

프로젝트 사이드바에서 **근태현황** 메뉴를 클릭하여 접근합니다.

### 8.2 근태 기록 추가

1. **근태 추가** 버튼을 클릭합니다.
2. **날짜**를 선택합니다.
3. **근태 유형**을 선택합니다.
4. 필요 시 **메모**를 입력합니다.
5. **저장** 버튼을 클릭합니다.

### 8.3 근태 유형

총 9종의 근태 유형을 지원합니다.

| 유형 | 설명 |
|------|------|
| **출근** | 정상 출근 |
| **연차** | 연차 휴가 |
| **오전반차** | 오전 반일 휴가 |
| **오후반차** | 오후 반일 휴가 |
| **병가** | 질병으로 인한 휴가 |
| **출장** | 외부 출장 |
| **지각** | 지각 |
| **조퇴** | 조기 퇴근 |
| **결근** | 무단 결근 |

### 8.4 월별 캘린더 뷰

캘린더 형태로 해당 월의 근태 현황을 한눈에 확인할 수 있습니다. 각 날짜 셀에 해당일의 근태 유형이 색상으로 표시되며, 셀을 클릭하면 상세 내역을 확인하거나 수정할 수 있습니다.

### 8.5 멤버별 필터링

상단의 멤버 선택 드롭다운에서 특정 멤버를 선택하면 해당 멤버의 근태 기록만 필터링하여 표시합니다. **전체**를 선택하면 모든 멤버의 근태를 확인할 수 있습니다.

### 8.6 월별 요약 테이블

페이지 하단에 월별 근태 요약 테이블이 표시됩니다. 각 멤버별로 출근, 연차, 반차, 병가, 출장, 지각, 조퇴, 결근 일수를 집계하여 보여줍니다.

## 9. 주간보고

프로젝트의 주간 진행 현황을 요약하고 내보낼 수 있는 기능입니다.

### 9.1 주간보고 접근

대시보드 또는 WBS 페이지 상단의 **주간보고** 버튼을 클릭하면 주간보고 미리보기 모달이 열립니다.

### 9.2 개요 탭

주간보고 모달의 **개요** 탭에서는 다음 정보를 확인할 수 있습니다:

| 항목 | 설명 |
|------|------|
| **KPI 카드** | 전체작업 수, 완료 작업 수, 공정율, 지연 작업 수 |
| **전주 대비 변화** | 각 KPI의 전주 대비 증감 표시 (델타값) |
| **담당자별 통계** | 담당자별 작업 현황 요약 |
| **근태현황** | 금주 멤버 근태 요약 |

### 9.3 상세 탭

**상세** 탭에서는 작업 목록을 섹션별로 확인할 수 있습니다:

| 섹션 | 설명 |
|------|------|
| **금주실적** | 금주에 진행된 작업 목록 |
| **금주완료** | 금주에 완료된 작업 목록 |
| **차주계획** | 다음 주에 진행 예정인 작업 목록 |
| **지연작업** | 현재 지연 중인 작업 목록 |

### 9.4 주차 네비게이션

모달 상단의 **◀** / **▶** 버튼으로 이전 주 또는 다음 주의 보고서를 조회할 수 있습니다. 스냅샷이 저장된 주차의 보고서 데이터를 불러옵니다.

### 9.5 엑셀 다운로드

**엑셀 다운로드** 버튼을 클릭하면 주간보고 내용을 엑셀(.xlsx) 파일로 내보냅니다. KPI 요약, 작업 목록, 담당자별 통계, 근태현황이 시트에 포함됩니다.

### 9.6 PPT 다운로드

**PPT 다운로드** 버튼을 클릭하면 주간보고를 파워포인트(.pptx) 파일로 내보냅니다. 생성되는 슬라이드 구성은 다음과 같습니다:

| 슬라이드 | 내용 |
|----------|------|
| **슬라이드 1** | 요약 현황 — KPI 카드, 프로그레스 바, 이슈/리스크 목록 |
| **슬라이드 2+** | 상세 작업 — 금주실적, 차주계획 작업 목록 |
| **슬라이드 N** | 근태현황 — 금주 + 차주 통합 근태 요약 (요일 + 날짜 표시) |

## 11. 계정 설정

개인 계정 정보를 관리하는 페이지입니다.

### 11.1 접근 방법

헤더 우측의 **사용자 메뉴**를 클릭한 후 **계정 설정**을 선택합니다.

### 11.2 이름 변경

1. 계정 설정 페이지에서 **이름** 입력란의 값을 수정합니다.
2. **저장** 버튼을 클릭하여 변경을 반영합니다.

### 11.3 비밀번호 변경

1. **현재 비밀번호**를 입력합니다.
2. **새 비밀번호** (6자 이상)를 입력합니다.
3. **비밀번호 확인**란에 새 비밀번호를 다시 입력합니다.
4. **비밀번호 변경** 버튼을 클릭합니다.

### 11.4 회원 탈퇴

1. 계정 설정 페이지 하단의 **회원 탈퇴** 버튼을 클릭합니다.
2. 확인 대화상자에서 탈퇴를 승인합니다.

> ⚠️ 회원 탈퇴 시 소유하고 있는 프로젝트가 **일괄 삭제**됩니다. 탈퇴 전에 필요한 데이터를 반드시 백업하세요.

## 15. 데이터 관리 (가져오기/내보내기)

### 15.1 WBS 엑셀 내보내기

**방법 1**: WBS 페이지 → **엑셀 다운로드** 버튼
**방법 2**: 설정 페이지 → **WBS 엑셀 내보내기** 버튼

내보내기 파일에는 다음 정보가 포함됩니다:
- 작업 계층 구조 (Phase/Activity/Task/Function)
- 작업명, 산출물, 담당자
- 가중치, 계획/실적 일정, 계획/실적 공정율
- 작업 상태

### 15.2 엑셀 가져오기

설정 페이지 → **엑셀 가져오기** 버튼으로 `.xlsx` 또는 `.xls` 파일을 업로드하여 WBS 데이터를 불러올 수 있습니다.

> ⚠️ 주의: 기존 작업이 있는 경우 덮어쓰기 여부를 확인합니다.

### 15.3 간트 차트 엑셀 내보내기

간트 차트 페이지 → **엑셀 다운로드** 버튼으로 현재 필터 기준의 간트 데이터를 별도의 엑셀로 내보낼 수 있습니다.

### 15.4 현황 보고서 (Word)

대시보드 → **현황 보고서** 버튼으로 프로젝트 현황을 Word(.docx) 문서로 자동 생성하여 다운로드합니다.

### 15.5 주간보고 엑셀 내보내기

주간보고 모달 → **엑셀 다운로드** 버튼으로 주간보고 내용을 엑셀(.xlsx) 파일로 내보냅니다. KPI 요약, 작업 목록, 담당자별 통계, 근태현황이 포함됩니다.

### 15.6 주간보고 PPT 내보내기

주간보고 모달 → **PPT 다운로드** 버튼으로 주간보고를 파워포인트(.pptx) 프레젠테이션으로 내보냅니다. 요약 현황, 상세 작업, 근태현황 슬라이드가 자동 생성됩니다.

### 15.7 프로젝트 복제

프로젝트 설정 페이지에서 **프로젝트 복제** 기능을 사용하면 기존 프로젝트의 WBS 구조와 멤버 구성을 복제하여 새 프로젝트를 생성할 수 있습니다.

## 17. 자주 묻는 질문 (FAQ)

### Q1. 로컬 모드와 온라인 모드의 차이는?

| 항목 | 로컬 모드 | 온라인 모드 (Supabase) |
|------|----------|----------------------|
| 계정 | 불필요 | 이메일/비밀번호 필요 |
| 데이터 저장 | 브라우저 로컬 | 서버(Supabase) |
| 멀티 사용자 | ❌ | ✅ |
| 데이터 동기화 | ❌ | ✅ |

### Q2. 작업이 자동으로 저장되나요?

WBS 페이지에서의 변경사항은 **약 0.7초 후 자동 저장**됩니다. 프로젝트 설정에서의 기본 정보 변경은 **저장** 버튼을 눌러야 합니다.

### Q3. 상위 작업의 공정율은 어떻게 계산되나요?

상위 작업의 공정율은 **하위 작업의 가중치와 공정율을 조합**하여 자동 계산됩니다.

### Q4. 엑셀 가져오기 시 기존 데이터는 어떻게 되나요?

기존 작업이 있는 경우 **덮어쓰기 확인** 대화상자가 표시됩니다. "확인"을 클릭하면 기존 데이터가 교체됩니다.

### Q5. 프로젝트 삭제 후 복구할 수 있나요?

아니요. 프로젝트 삭제는 **영구적**이며 되돌릴 수 없습니다. 삭제 전에 WBS 엑셀 내보내기로 데이터를 백업해두는 것을 권장합니다.

### Q6. 관리 기능 (상태 변경, 삭제)에 접근할 수 없어요.

해당 기능은 **관리자** 권한이 필요합니다. 시스템 관리자에게 권한 부여를 요청하세요.

### Q7. 주간보고는 어떻게 만드나요?

대시보드 또는 WBS 페이지 상단의 **주간보고** 버튼을 클릭하면 주간보고 미리보기 모달이 열립니다. 모달에서 엑셀 또는 PPT로 다운로드할 수 있습니다.

### Q8. 근태 기록은 어디서 관리하나요?

프로젝트 사이드바의 **근태현황** 메뉴에서 근태를 기록하고 관리할 수 있습니다.

### Q9. 회원가입 후 바로 사용할 수 있나요?

온라인 모드(Supabase)에서는 **관리자 승인**이 필요합니다. 회원가입 후 관리자가 승인을 완료하면 정상적으로 이용할 수 있습니다. 로컬 모드에서는 승인 없이 즉시 사용할 수 있습니다.

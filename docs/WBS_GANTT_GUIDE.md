## 1. 개요

DK Flow는 프로젝트의 작업 구조를 **WBS(Work Breakdown Structure) 테이블**로 편집하고, **간트 차트**로 일정을 시각화하는 두 개의 핵심 화면을 제공합니다.

| 화면 | 경로 | 핵심 역할 |
|------|------|-----------|
| **WBS** | `/projects/:projectId/wbs` | 작업 생성·편집·삭제, 계층 구조 관리, 자동화 |
| **간트 차트** | `/projects/:projectId/gantt` | 타임라인 시각화, 일정 검색·필터, 빠른 편집 |

두 화면은 **동일한 데이터(taskStore)**를 공유하므로, 한쪽에서 수정한 내용은 다른쪽에 즉시 반영됩니다.

## 2. WBS 작성

### 2.1 WBS 화면 구성

WBS 페이지는 크게 **상단 도구 모음**과 **작업 테이블** 두 영역으로 구성됩니다.

#### 상단 도구 모음

| 버튼 | 기능 |
|------|------|
| **Phase 추가** | 최상위(1레벨) 작업을 새로 추가 |
| **빠른 초안 생성** | 프로젝트 설명을 입력하면 키워드 기반으로 적합한 템플릿을 자동 선택하여 WBS 초안을 생성 |
| **템플릿 적용** | 미리 정의된 템플릿 중 하나를 선택하여 전체 WBS 구조를 한 번에 생성 |
| **일정 자동 계산** | 작업 기간과 선후행 관계를 기반으로 계획 일정을 자동 배치 |
| **선후행 연결** | 같은 계층의 leaf 작업끼리 순차적 종속 관계를 자동 생성 |
| **자동채움** | 빈 산출물 추천, 빈 담당자 자동 배정, 가중치 재계산을 일괄 실행 |
| **주간보고** | 현재 WBS 데이터 기반으로 주간보고서를 생성하는 모달 열기 |
| **전체 펼침 / 전체 접기** | 모든 작업의 하위 항목을 펼치거나 접음 |
| **엑셀 다운로드** | WBS 테이블을 Excel(.xlsx) 파일로 내보내기 |
| **저장** | 변경사항을 즉시 저장 (자동 저장과 별도로 수동 저장 가능) |
| **실행취소 / 재실행** | 최대 50단계까지 Undo/Redo 지원 |

#### 작업 테이블 컬럼

| 컬럼 | 설명 | 편집 방식 |
|------|------|-----------|
| **펼침/접기** | 하위 작업이 있는 경우 접기/펼치기 토글 | 클릭 |
| **구분** | 작업 레벨 표시 (Phase / Activity / Task / Function) | 자동 (레벨에 따라 결정) |
| **작업명** | 작업의 이름 | 클릭 → 텍스트 입력 |
| **산출물** | 작업의 결과물 | 클릭 → 텍스트 입력 |
| **담당자** | 작업 담당 멤버 | 드롭다운 선택 (새 멤버 즉석 생성 가능) |
| **가중치** | 작업의 상대적 비중 (0~1 사이 소수점) | 클릭 → 숫자 입력 |
| **계획 시작 / 종료** | 예정 일정 | 날짜 선택기 |
| **실적 시작 / 종료** | 실제 수행 일정 | 날짜 선택기 |
| **계획 공정율** | 계획 대비 진척률 (%) | 클릭 → 숫자 입력 (0~100) |
| **실적 공정율** | 실제 진척률 (%) | 클릭 → 숫자 입력 (0~100) |
| **상태** | 대기 / 진행중 / 완료 / 보류 | 드롭다운 선택 |
| **작업** | 하위 작업 추가(+), 삭제(휴지통) 버튼 | 클릭 |

### 2.2 작업 추가 방법

DK Flow는 세 가지 방식으로 작업을 추가할 수 있습니다.

#### 방법 1: 수동 추가

1. **Phase 추가** 버튼을 클릭하면 최상위(Level 1) 작업이 추가됩니다.
2. 각 행의 **+** 버튼을 클릭하면 해당 작업의 하위 작업이 추가됩니다.
3. 새로 추가된 작업의 **작업명** 셀이 자동으로 편집 모드로 활성화됩니다.
4. 작업명을 입력한 후 **Enter** 키 또는 셀 바깥을 클릭하여 확정합니다.

계층 구조는 다음과 같이 최대 4단계까지 지원됩니다:

```
Level 1: Phase (단계)
  └─ Level 2: Activity (활동)
       └─ Level 3: Task (작업)
            └─ Level 4: Function (기능)
```

#### 방법 2: 템플릿 적용

미리 정의된 프로젝트 유형별 템플릿을 사용하면 전체 WBS 골격을 한 번에 만들 수 있습니다.

1. **템플릿 적용** 버튼을 클릭합니다.
2. 템플릿 목록에서 프로젝트 유형에 맞는 것을 선택합니다.

| 템플릿 ID | 이름 | 설명 | 대상 |
|-----------|------|------|------|
| `web-launch` | Website Launch | 기업 사이트, 랜딩 페이지, 쇼핑몰 출시 | 마케팅 사이트 및 리뉴얼 프로젝트 |
| `mobile-app` | Mobile App | 모바일 앱 개발 및 스토어 출시 | iOS/Android 앱 프로젝트 |
| `internal-system` | Internal System | ERP, 백오피스, 어드민 등 내부 시스템 | 업무 시스템 구축 프로젝트 |

3. **적용** 버튼을 클릭하면 현재 WBS에 템플릿 구조가 반영됩니다.
4. 프로젝트 시작일이 설정되어 있으면 자동으로 일정이 배치됩니다.

> **주의**: 템플릿을 적용하면 기존 작업이 모두 대체됩니다. 필요한 경우 먼저 엑셀로 백업하세요.

#### 방법 3: 빠른 초안 생성 (프롬프트 기반)

프로젝트 설명만 입력하면 키워드를 분석하여 가장 적합한 템플릿을 자동 선택하고, 추가 작업을 보강한 초안을 만들어줍니다.

1. **빠른 초안 생성** 버튼을 클릭합니다.
2. 프로젝트 설명을 입력합니다. 예시:
   - `"웹 사이트 리뉴얼 프론트엔드 개발"` → web-launch 템플릿 + 프론트엔드 추가 작업
   - `"모바일 앱 출시 Android"` → mobile-app 템플릿 + Android 관련 작업
   - `"ERP 내부 업무시스템 구축"` → internal-system 템플릿
3. 키워드 분석 결과와 선택된 템플릿, 매칭된 키워드가 표시됩니다.
4. **적용** 버튼으로 초안을 WBS에 반영합니다.

**키워드 매핑 규칙**:

| 키워드 | 매핑 템플릿 |
|--------|------------|
| 웹, 홈페이지, 사이트, 쇼핑몰, 리뉴얼, 랜딩, 브랜드 | Website Launch |
| 앱, iOS, Android, 모바일, 스토어, 앱출시 | Mobile App |
| ERP, 백오피스, 내부, 어드민, 관리시스템, 업무시스템, 운영시스템 | Internal System |

추가로 다음 키워드가 포함되면 관련 작업이 보강됩니다:
- **디자인/UX/UI/와이어/시안** → Design review 작업 추가
- **프론트/frontend/React/화면** → Frontend integration polish 작업 추가
- **백엔드/backend/서버/API** → API optimization 작업 추가
- **테스트/QA/검증/보안** → Security review 작업 추가
- **배포/DevOps/CI/CD/인프라** → Infrastructure audit 작업 추가

### 2.3 인라인 편집

WBS 테이블의 각 셀을 **직접 클릭**하여 편집할 수 있습니다.

#### 텍스트 필드 (작업명, 산출물)

1. 셀을 클릭하면 입력 상자로 전환됩니다.
2. 값을 입력합니다.
3. **Enter** 키 또는 셀 바깥 클릭으로 확정합니다.
4. **Escape** 키로도 편집을 종료할 수 있습니다.

#### 숫자 필드 (가중치, 계획 공정율, 실적 공정율)

1. 셀을 클릭하면 숫자 입력 상자로 전환됩니다.
2. 공정율은 0~100 사이 정수만 입력 가능합니다.
3. 가중치는 소수점 입력이 가능합니다 (예: 0.125).

#### 날짜 필드 (계획 시작/종료, 실적 시작/종료)

- 날짜 선택기가 항상 표시되어 있습니다.
- 날짜를 선택하면 즉시 반영됩니다.
- 날짜를 비우려면 선택된 값을 지우면 됩니다.

#### 상태 필드

드롭다운에서 다음 중 선택합니다:

| 상태 | 한국어 레이블 | 설명 |
|------|-------------|------|
| `pending` | 대기 | 아직 시작 전 |
| `in_progress` | 진행중 | 현재 수행 중 |
| `completed` | 완료 | 작업 완료됨 |
| `on_hold` | 보류 | 일시 중단 |

#### 담당자 필드

- 드롭다운에서 기존 멤버를 선택합니다.
- **새 멤버 생성**: 드롭다운 하단에서 이름을 직접 입력하여 새 멤버를 즉석에서 추가할 수 있습니다. 추가된 멤버는 프로젝트 멤버 목록에도 자동 반영됩니다.

### 2.4 드래그 & 드롭으로 작업 이동

작업 행을 드래그하여 순서를 변경하거나 다른 작업의 하위로 이동할 수 있습니다.

#### 조작 방법

1. 행 왼쪽의 **드래그 핸들(⋮⋮)** 아이콘을 잡고 드래그합니다.
2. 드래그 중 대상 행의 위치에 따라 삽입 위치가 결정됩니다:

| 마우스 위치 | 동작 | 시각적 표시 |
|-------------|------|------------|
| 행의 **상단 25%** | 대상 행 **앞**에 삽입 | 상단 파란 라인 |
| 행의 **중앙 50%** | 대상 행의 **하위 작업**으로 이동 | 행 전체 하이라이트 |
| 행의 **하단 25%** | 대상 행 **뒤**에 삽입 | 하단 파란 라인 |

3. 드롭하면 작업이 해당 위치로 이동합니다.

#### 제약 사항

- **자기 자신의 하위로 이동 불가**: 부모 작업을 자신의 자녀/손자 작업 아래로 이동할 수 없습니다 (순환 참조 방지).
- **레벨 4(Function) 하위로 이동 불가**: 최대 4단계까지만 지원하므로, Level 4 작업의 하위로는 이동할 수 없습니다.

### 2.5 자동화 기능

WBS 작성을 효율화하는 여러 자동화 기능을 제공합니다.

#### 2.5.1 일정 자동 계산

**버튼**: `일정 자동 계산`

작업의 기간(`durationDays`)과 선후행 관계(`predecessorIds`)를 기반으로 계획 일정(`planStart`, `planEnd`)을 자동 배치합니다.

**동작 원리**:
1. 각 작업의 기간(일)을 확인합니다. 기간이 없으면 기본 2일을 사용합니다.
2. 선행 작업이 있으면 선행 작업 종료일 다음 날부터 시작합니다.
3. 선행 작업이 없으면 프로젝트 시작일(또는 현재일)부터 시작합니다.
4. leaf 작업의 일정이 계산되면, 부모 작업의 일정은 자식들의 최소 시작일~최대 종료일로 자동 집계됩니다.

#### 2.5.2 선후행 연결

**버튼**: `선후행 연결`

같은 부모 아래의 leaf 작업(하위 자녀가 없는 작업)들을 **순서대로 연결**합니다.

**예시**: 같은 Activity 아래에 Task A, Task B, Task C가 있으면
- Task B의 선행 작업 = Task A
- Task C의 선행 작업 = Task B

이후 `일정 자동 계산`을 실행하면 연결된 순서에 맞게 일정이 배치됩니다.

#### 2.5.3 자동채움

**버튼**: `자동채움` (마법봉 아이콘)

빈 필드를 지능적으로 채워줍니다. 세 가지 기능을 일괄 실행합니다:

**산출물 자동 추천**

작업명의 키워드를 분석하여 적합한 산출물을 자동으로 추천합니다. 산출물이 이미 있는 작업은 건너뜁니다.

| 키워드 예시 | 추천 산출물 |
|-------------|-----------|
| 요구사항, requirement | 요구사항 정의서 |
| 와이어프레임, wireframe | 와이어프레임 |
| 디자인, design, UI | 디자인 시안 |
| API, api, 인터페이스 개발 | API 명세서 |
| DB, database, ERD | ERD / 스키마 문서 |
| 테스트, test, QA | 테스트 결과서 |
| 배포, deploy, CI/CD | 배포 스크립트 |
| 매뉴얼, manual, 가이드 | 사용자 매뉴얼 |
| 검수, 인수, 검증 | 검수 확인서 |
| 출시, 런칭, launch | 출시 체크리스트 |

**담당자 자동 배정**

담당자가 지정되지 않은 leaf 작업에 멤버를 **라운드 로빈** 방식으로 자동 배정합니다. 프로젝트 멤버 목록에서 순서대로 배분합니다.

**가중치 재계산**

leaf 작업의 가중치를 작업 수에 비례하여 재계산합니다. 전체 가중치의 합이 1.0이 되도록 정규화합니다.

#### 2.5.4 필드 자동 연동 (상태 ↔ 진행률 ↔ 날짜)

leaf 작업(자녀가 없는 최하위 작업)의 특정 필드를 변경하면 관련 필드가 자동으로 동기화됩니다.

| 변경한 필드 | 자동 동기화 내용 |
|-------------|----------------|
| **상태** → `진행중` | 실적 시작일이 비어있으면 오늘 날짜로 설정 |
| **상태** → `완료` | 실적 공정율 100%, 실적 종료일이 비어있으면 오늘로 설정 |
| **상태** → `대기` | 실적 공정율 0%, 실적 시작일/종료일 초기화 |
| **실적 공정율** → `100` | 상태를 `완료`로 변경, 실적 종료일이 비어있으면 오늘로 설정 |
| **실적 공정율** → `1~99` | 상태를 `진행중`으로 변경, 실적 시작일이 비어있으면 오늘로 설정 |
| **실적 공정율** → `0` | 상태를 `대기`로 변경 |
| **실적 시작일** 입력 | 상태가 `대기`이면 `진행중`으로 변경 |
| **실적 종료일** 입력 | 상태를 `완료`로 변경, 실적 공정율 100% |

> **참고**: 부모 작업의 일정, 진행률, 상태는 항상 자식 작업에서 자동 집계됩니다. 부모 작업을 직접 수정해도 다음 정규화(저장) 시점에 자식 기반 값으로 덮어써집니다.

---

### 2.6 주간보고서 생성

WBS 데이터를 기반으로 주간보고서를 생성할 수 있습니다.

1. **주간보고** 버튼을 클릭합니다.
2. 보고서 모달이 열리며 기본적으로 현재 주차의 보고서가 표시됩니다.
3. **좌/우 화살표**로 기준 주차를 앞뒤로 이동할 수 있습니다.

**보고서 내용**:
- **금주 실적**: 이번 주에 수행된 작업 목록 (담당자, 상태, 진척률 포함)
- **차주 계획**: 다음 주에 예정된 작업 목록
- **지연 작업**: 계획 대비 지연되고 있는 작업 (지연 일수 표시)
- **이슈 요약**: 보류 중이거나 주의가 필요한 사항

**스냅샷 기능**:
- **저장** 버튼으로 현재 주차의 보고서 스냅샷을 localStorage에 저장합니다.
- 같은 주차에 다시 저장하면 덮어쓰기됩니다.
- 이전 주차의 스냅샷과 비교하여 변화를 확인할 수 있습니다.

**Excel 내보내기**:
- 보고서 모달에서 **다운로드** 버튼을 클릭하면 서식이 적용된 Excel 파일로 내보낼 수 있습니다.
- Pretendard 폰트, 상태별 색상 코드, 섹션 구분 등 전문적인 포맷이 적용됩니다.

---

### 2.7 내보내기

#### WBS 엑셀 다운로드

- **엑셀 다운로드** 버튼을 클릭합니다.
- 전체 WBS 테이블이 ExcelJS 기반으로 서식이 적용된 `.xlsx` 파일로 다운로드됩니다.
- 프로젝트명, 작업 계층 구조, 일정, 진척률, 상태 등 모든 정보가 포함됩니다.

### 2.8 저장 및 실행취소

#### 자동 저장

- 작업을 변경하면 **700ms 디바운스** 후 자동으로 저장됩니다.
- 저장 상태가 상단 도구 모음에 표시됩니다:
  - `변경사항 저장 대기` — 변경이 감지되어 디바운스 중
  - `저장중...` — 저장 진행 중 (로딩 스피너)
  - `HH:mm 저장됨` — 마지막 저장 시각
  - `저장 실패` — 오류 발생 시

#### 수동 저장

- **저장** 버튼 또는 `Ctrl+S` / `Cmd+S`로 즉시 저장합니다.

#### 실행취소 / 재실행

- **Undo**: `Ctrl+Z` / `Cmd+Z` 또는 ↩ 버튼 (최대 50단계)
- **Redo**: `Ctrl+Y` / `Cmd+Y` 또는 ↪ 버튼

> **참고**: 셀 입력 중에는 히스토리가 기록되지 않으며, 셀을 떠날 때(blur) 한 번에 기록됩니다. 이를 통해 글자 하나마다 히스토리가 쌓이는 것을 방지합니다.

## 3. 간트 차트

### 3.1 간트 차트 화면 구성

간트 차트 페이지는 상단의 **요약 및 포커스 카드**, 중간의 **검색/필터 도구 모음**, 하단의 **간트 본체** 세 영역으로 구성됩니다.

#### 상단 영역 (2칼럼 그리드)

| 왼쪽: 프로젝트 히어로 카드 | 오른쪽: 작업 포커스 카드 |
|---------------------------|------------------------|
| 프로젝트명 + 일정 흐름 제목 | 선택된 작업의 상세 정보 |
| 표시 작업 수, 오픈 작업 수, 지연 작업 수 | 레벨, 작업명, 담당자, 상태 |
| | 계획 기간, 실적 기간, 예상 기간, 지연일 |
| | 실적 공정율 프로그레스 바 |
| | 산출물 정보 |
| | **빠른 편집** 폼 |

#### 중간 도구 모음

- 검색 입력창 (작업명, 산출물, 담당자)
- 상태 필터 버튼 (전체 / 진행중·대기 / 지연 / 완료)
- 엑셀 다운로드, 저장 버튼
- 보기 범위 선택 (6주 / 12주 / 24주)
- 행 밀도 (Compact / Comfortable)
- 주말 강조 토글

#### 하단 간트 본체 (좌우 분할)

| 왼쪽: 작업 목록 (360px 고정폭) | 오른쪽: 간트 차트 바 |
|-------------------------------|---------------------|
| 트리 구조 작업 목록 | 날짜 헤더 (주 단위) |
| 레벨 표시, 접기/펼치기 | 계획 바 (파란색 그라디언트) |
| 일정 범위 텍스트 | 실적 바 (녹색 그라디언트) |
| 진척률, 지연 배지 | 오늘 날짜 빨간 세로선 |

두 패널의 **세로 스크롤이 동기화**됩니다. 왼쪽을 스크롤하면 오른쪽도 따라 움직이고, 그 반대도 마찬가지입니다.

---

### 3.2 차트 읽는 법

간트 차트에서 사용되는 시각 요소:

| 요소 | 색상/모양 | 의미 |
|------|----------|------|
| **계획 바** | 파란-청록 그라디언트 (`#155e75` → `#1f8f86`) | 계획 시작일~계획 종료일 |
| **실적 바** | 녹색 그라디언트 (`#1fa37a` → `#34c997`) | 실적 시작일~실적 종료일 |
| **빨간 세로선** | 빨간색 | 오늘 날짜 |
| **주말 영역** | 연한 배경색 | 토요일·일요일 (주말 강조 켜진 경우) |
| **+Nd 배지** | 빨간 배지 | 지연 일수 (N일 지연) |
| **행 하이라이트** | 연한 청록 배경 | 현재 선택된 작업 |
| **Level 1 행** | 약간 진한 배경 + 볼드 | Phase(단계) 수준 작업 |

**날짜 헤더**: 상단에 주(week) 단위로 날짜가 표시됩니다. 각 셀은 하루를 나타냅니다.

**자동 포커스**: 작업을 선택하면 해당 작업의 일정 범위가 보이도록 차트가 자동으로 스크롤됩니다. 현재 뷰 범위 밖에 있는 작업을 선택하면 시작점이 자동으로 조정됩니다.

---

### 3.3 작업 선택 및 빠른 편집

#### 작업 선택

- 왼쪽 작업 목록 또는 오른쪽 차트 바를 클릭하면 해당 작업이 선택됩니다.
- 선택된 작업의 상세 정보가 오른쪽 상단 **작업 포커스 카드**에 표시됩니다.
- 페이지 진입 시 자동 선택: 지연 작업 → 미완료 작업 → 첫 번째 작업 순서로 자동 선택됩니다.

#### 빠른 편집

작업 포커스 카드 하단에 **빠른 편집** 폼이 제공됩니다. WBS 페이지로 이동하지 않고도 핵심 정보를 수정할 수 있습니다.

편집 가능한 필드:

| 필드 | 입력 방식 |
|------|----------|
| 작업명 | 텍스트 입력 |
| 산출물 | 텍스트 입력 |
| 담당자 | 드롭다운 선택 |
| 상태 | 드롭다운 선택 (대기/진행중/완료/보류) |
| 계획 시작 / 종료 | 날짜 선택기 |
| 실적 시작 / 종료 | 날짜 선택기 |
| 실적 공정율 | 숫자 입력 (0~100) |
| 계획 공정율 | 숫자 입력 (0~100) |

수정 사항은 즉시 반영되며, 자동 저장됩니다.

---

### 3.4 검색 및 필터

#### 텍스트 검색

- 검색창에 키워드를 입력하면 **작업명**, **산출물**, **담당자명**을 대상으로 실시간 검색합니다.
- 검색 결과에 해당하는 작업과 그 **상위 부모 작업**이 함께 표시됩니다 (계층 구조 유지).

#### 상태 필터

| 필터 | 표시 대상 |
|------|----------|
| **전체** | 모든 작업 |
| **진행중/대기** | 상태가 `completed`가 아닌 작업 |
| **지연** | 계획 종료일 대비 지연되고 있는 작업 |
| **완료** | 상태가 `completed`인 작업 |

검색과 필터는 **동시에 적용**됩니다. 예: "디자인"을 검색하고 "지연" 필터를 선택하면, 이름/산출물/담당자에 "디자인"이 포함되면서 지연 중인 작업만 표시됩니다.

---

### 3.5 보기 옵션

#### 보기 범위

차트에 표시할 기간을 조절합니다:

| 옵션 | 설명 |
|------|------|
| **6주** | 단기 집중 뷰 — 세부 일정을 확대하여 확인 |
| **12주** (기본값) | 표준 뷰 — 약 3개월 범위 |
| **24주** | 장기 뷰 — 약 6개월 범위, 전체 프로젝트 일정 조망 |

#### 행 밀도

| 옵션 | 행 높이 | 하루 너비 | 용도 |
|------|--------|----------|------|
| **Compact** | 34px | 32px | 많은 작업을 한 화면에 보고 싶을 때 |
| **Comfortable** (기본값) | 42px | 44px | 여유 있는 간격으로 가독성 향상 |

#### 주말 강조

- 켜기(기본): 토요일·일요일 컬럼에 연한 배경색 적용
- 끄기: 주말 구분 없이 동일하게 표시

#### 차트 네비게이션

차트 상단의 **좌/우 화살표** 버튼으로 시작 날짜를 1주씩 앞뒤로 이동할 수 있습니다. **오늘로 이동** 버튼을 클릭하면 현재 날짜 기준으로 차트가 리셋됩니다.

---

### 3.6 전체화면 보기

간트 본체 우측 상단의 **크게 보기** 버튼(↗)을 클릭하면 전체화면 모달로 간트 차트를 확대합니다. 모달 안에서도 동일한 인터랙션(클릭 선택, 스크롤)이 가능합니다. **ESC** 키 또는 닫기 버튼으로 모달을 닫습니다.

---

### 3.7 간트 차트 내보내기

**엑셀 다운로드** 버튼을 클릭하면 현재 필터/검색이 적용된 작업 목록을 Excel 파일로 내보냅니다. 내보내기에는 다음 정보가 포함됩니다:
- 프로젝트명, 적용된 필터 레이블, 검색 키워드
- 작업 목록 (레벨, 작업명, 담당자, 상태, 진척률)
- 주 단위 타임라인 데이터

## 4. 작업 계층 구조 이해하기

DK Flow의 작업은 4단계 계층으로 구성됩니다:

```
Phase (Level 1) — 프로젝트의 큰 단계
│
├── Activity (Level 2) — 단계 내의 주요 활동
│   │
│   ├── Task (Level 3) — 구체적인 작업 항목
│   │   │
│   │   ├── Function (Level 4) — 가장 세부적인 기능 단위
│   │   └── Function (Level 4)
│   │
│   └── Task (Level 3)
│
└── Activity (Level 2)
```

### 부모-자식 관계의 핵심 규칙

| 항목 | 규칙 |
|------|------|
| **일정 (계획 시작/종료)** | 부모 = 자식들의 최소 시작일 ~ 최대 종료일 |
| **실적 일정** | 부모 = 자식들의 최소 실적 시작일 ~ 최대 실적 종료일 |
| **진행률 (계획/실적)** | 부모 = 자식들의 가중 평균 (가중치 기반) |
| **상태** | 모든 자식이 완료 → 부모 완료, 하나라도 진행중 → 부모 진행중 |
| **가중치** | 부모 = 자식들의 가중치 합계 |

> **중요**: 부모 작업의 위 속성들은 항상 자식으로부터 자동 계산됩니다. 부모를 직접 수정해도 다음 정규화(저장) 시점에 자식 기반 값으로 되돌아갑니다. **실제 데이터를 입력하는 대상은 항상 leaf 작업(최하위 작업)**입니다.

---

## 5. 필드 자동 연동 규칙

leaf 작업의 필드 간 자동 연동을 상세히 정리합니다.

### 상태 변경 시

```
상태 → 진행중 (in_progress)
  └─ 실적 시작일이 비어있으면 → 오늘 날짜 자동 설정

상태 → 완료 (completed)
  ├─ 실적 공정율 → 100%
  └─ 실적 종료일이 비어있으면 → 오늘 날짜 자동 설정

상태 → 대기 (pending)
  ├─ 실적 공정율 → 0%
  ├─ 실적 시작일 → 초기화 (null)
  └─ 실적 종료일 → 초기화 (null)
```

### 실적 공정율 변경 시

```
실적 공정율 → 100
  ├─ 상태 → 완료 (completed)
  └─ 실적 종료일이 비어있으면 → 오늘 날짜 자동 설정

실적 공정율 → 1~99
  ├─ 상태 → 진행중 (in_progress)
  └─ 실적 시작일이 비어있으면 → 오늘 날짜 자동 설정

실적 공정율 → 0
  └─ 상태 → 대기 (pending)
```

### 실적 날짜 변경 시

```
실적 시작일 입력
  └─ 상태가 대기(pending)이면 → 진행중 (in_progress)으로 변경

실적 종료일 입력
  ├─ 상태 → 완료 (completed)
  └─ 실적 공정율 → 100%
```

## 7. 실전 워크플로우 예시

### 7.1 새 프로젝트 WBS를 처음부터 빠르게 세팅하기

```
1. 프로젝트 생성 후 WBS 페이지 진입
2. [빠른 초안 생성] → 프로젝트 설명 입력 (예: "쇼핑몰 웹사이트 리뉴얼")
3. 초안 확인 후 [적용]
4. 불필요한 작업 삭제, 필요한 작업 추가로 조정
5. [선후행 연결] → 같은 레벨의 작업들에 순차 종속성 부여
6. [일정 자동 계산] → 선후행 관계 기반으로 계획 일정 배치
7. [자동채움] → 산출물 추천, 담당자 배정, 가중치 계산
8. 세부 조정 후 저장
```

### 7.2 기존 프로젝트에 작업 추가하기

```
1. WBS 페이지에서 추가할 위치의 상위 작업 행 찾기
2. 해당 행의 [+] 버튼 클릭 → 하위 작업 추가
3. 작업명, 담당자, 일정, 산출물 입력
4. 필요하면 드래그 & 드롭으로 위치 조정
5. Ctrl+S로 저장
```

### 7.3 주간 일정 점검 및 보고

```
1. 간트 차트 페이지 진입
2. [지연] 필터 선택 → 지연 작업 확인
3. 지연 작업 클릭 → 포커스 카드에서 상세 확인
4. 빠른 편집으로 일정/상태 조정
5. WBS 페이지로 이동 → [주간보고] 버튼 클릭
6. 주간보고서 확인 후 [스냅샷 저장] 또는 [엑셀 다운로드]
```

### 7.4 프로젝트 진행률 갱신

```
1. WBS 페이지에서 완료된 작업의 상태를 [완료]로 변경
   → 실적 공정율 100%, 실적 종료일이 자동 설정됨
2. 진행 중인 작업의 실적 공정율을 업데이트
   → 상태가 자동으로 [진행중]으로 변경됨
3. 부모 작업의 진행률은 자동 집계됨
4. 간트 차트에서 전체 타임라인 확인
```

## 8. 자주 묻는 질문 (FAQ)

### Q: 부모 작업의 진행률을 직접 수정했는데 원래 값으로 돌아갑니다.

**A**: 부모 작업의 일정, 진행률, 상태는 항상 자식 작업에서 자동 집계됩니다. 실제 데이터를 수정할 때는 최하위 작업(leaf task)에서 변경해야 합니다.

### Q: 템플릿을 적용하면 기존 작업이 사라지나요?

**A**: 네, 템플릿 적용 시 기존 WBS가 대체됩니다. 적용 전에 엑셀 다운로드로 백업하거나, 적용 후 즉시 `Ctrl+Z`로 실행취소할 수 있습니다.

### Q: 자동 저장이 되지 않는 것 같습니다.

**A**: 자동 저장은 변경 감지 후 700ms 대기 후 실행됩니다. 상단 도구 모음의 저장 상태 배지를 확인하세요. `저장 실패`가 표시되면 Supabase 연결 상태를 확인하세요. 로컬 모드에서는 localStorage에 저장됩니다.

### Q: 간트 차트에서 작업의 바가 보이지 않습니다.

**A**: 계획 시작일 또는 계획 종료일이 입력되지 않은 작업은 바가 표시되지 않습니다. WBS에서 해당 작업의 일정을 입력하거나, `일정 자동 계산` 기능을 사용하세요.

### Q: 드래그 & 드롭이 안 됩니다.

**A**: 행 왼쪽의 드래그 핸들(⋮⋮) 아이콘을 잡고 드래그해야 합니다. 행의 다른 부분을 잡으면 셀 편집 모드로 진입할 수 있습니다. 또한 자기 자신의 하위로는 이동할 수 없습니다.

### Q: 간트 차트와 WBS의 데이터는 동기화되나요?

**A**: 네, 두 페이지는 동일한 데이터 저장소(taskStore)를 사용합니다. 한쪽에서 수정한 내용은 다른 페이지로 이동 시 즉시 반영됩니다.

### Q: 주간보고서의 스냅샷은 어디에 저장되나요?

**A**: 프로젝트별로 localStorage에 저장됩니다 (`weekly_snapshots_{projectId}` 키). 같은 주차에 다시 저장하면 기존 스냅샷을 덮어씁니다. 브라우저 데이터를 삭제하면 스냅샷도 사라지므로, 중요한 보고서는 Excel로 내보내서 보관하세요.

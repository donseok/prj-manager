# PRD (Product Requirements Document)
# 프로젝트 관리 시스템 - 제품 요구사항 정의서

## 1. 개요

### 1.1 제품 비전
엑셀 기반 프로젝트 관리의 한계를 극복하고, 웹 기반으로 IT 프로젝트의 일정 및 작업을 관리할 수 있는 시스템 구축

### 1.2 목표
- 엑셀과 동등하거나 더 높은 자유도로 WBS 관리
- 실시간 협업 및 공유 가능
- 대시보드를 통한 프로젝트 현황 한눈에 파악
- 다양한 프로젝트 유형 지원

### 1.3 대상 사용자
- 프로젝트 관리자 (PM)
- 프로젝트 리더 (PL)
- 개발자/작업자
- 경영진/이해관계자

## 3. 비기능 요구사항

### 3.1 성능
- 페이지 로딩: 3초 이내
- 데이터 저장: 실시간 자동 저장
- 동시 사용자: 100명 이상

### 3.2 사용성
- 엑셀과 유사한 UX
- 키보드 단축키 지원
- 반응형 디자인 (데스크톱 우선)

### 3.3 보안
- 사용자 인증
- 프로젝트별 접근 권한

## 4. 제약사항
- 무료 인프라 사용 (Vercel, Supabase 무료 티어)

## 5. 용어 정의
| 용어 | 설명 |
|------|------|
| WBS | Work Breakdown Structure, 작업분류체계 |
| Phase | 프로젝트 단계 (분석, 설계, 개발 등) |
| Activity | 단계 내 활동 |
| Task | 구체적 작업 |
| Function | 세부 기능/산출물 |
| 가중치 | 작업의 상대적 중요도/비중 |
| 공정율 | 작업 진행률 (0~100%) |
| 달성율 | 계획 대비 실적 비율 |

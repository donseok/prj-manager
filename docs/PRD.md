# PRD (Product Requirements Document)
# DK Flow - 프로젝트 관리 시스템 제품 요구사항 정의서

**버전**: 3.0
**최종 수정일**: 2026-03-22
**제품명**: DK Flow (2026 Workspace Edition)

---

## 1. 개요

### 1.1 제품 비전
엑셀 기반 프로젝트 관리의 한계를 극복하고, 웹 기반으로 IT 프로젝트의 일정 및 작업을 관리할 수 있는 시스템. WBS 계층 관리, 간트 차트 시각화, 대시보드 분석을 핵심으로 한다.

### 1.2 목표
- 엑셀과 동등하거나 더 높은 자유도로 WBS 관리
- 실시간 협업 및 공유 가능
- 대시보드를 통한 프로젝트 현황 한눈에 파악
- 다양한 프로젝트 유형 지원
- 다크모드 지원으로 작업 환경 최적화
- AI 챗봇을 통한 프로젝트 현황 조회

### 1.3 대상 사용자
- 프로젝트 관리자 (PM)
- 프로젝트 리더 (PL)
- 개발자/작업자
- 경영진/이해관계자
- 시스템 관리자 (Admin)

## 3. 비기능 요구사항

### 3.1 성능
- 페이지 로딩: 3초 이내
- 데이터 저장: 디바운스 자동 저장 (700ms)
- Undo/Redo 히스토리: 50건 유지

### 3.2 사용성
- 엑셀과 유사한 인라인 편집 UX
- 키보드 단축키 지원 (Ctrl+Z/Y)
- 반응형 디자인 (데스크톱 우선)
- 한국어 UI (날짜 포맷 포함)

### 3.3 보안
- 사용자 인증 (Supabase Auth 또는 로컬)
- 프로젝트별 접근 권한 (RLS)
- 관리자 전용 라우트 보호

### 3.4 호환성
- 최신 Chrome, Firefox, Safari, Edge 지원
- 프론트엔드 전용 모드 (Supabase 없이 localStorage 동작)

---

## 4. 제약사항
- 무료 인프라 사용 (Vercel, Supabase 무료 티어)
- React 19 기반 SPA
- Supabase 미설정 시 localStorage 폴백
- 한국어 전용 UI

## 5. 용어 정의
| 용어 | 설명 |
|------|------|
| WBS | Work Breakdown Structure, 작업분류체계 |
| Phase | 프로젝트 단계 (Level 1) - 분석, 설계, 개발 등 |
| Activity | 단계 내 활동 (Level 2) |
| Task | 구체적 작업 (Level 3) |
| Function | 세부 기능/산출물 (Level 4) |
| 가중치 | 작업의 상대적 중요도/비중 (weight) |
| 공정율 | 작업 진행률 (0~100%) |
| 달성율 | 계획 대비 실적 비율 |
| 진척기준일 | 공정율 비교 기준 날짜 (baseDate) |
